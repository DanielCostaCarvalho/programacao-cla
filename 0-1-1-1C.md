# Criação de um site simples usando websocket CORREÇÃO
## Descrição
Esse foi um desafio aberto a várias formas de se resolver os requisitos funcionais, então serão apontados caminhos, mas outros não apontados aqui são possíveis.

O objetivo do desafio foi mostrar na prática o funcionamento de um site usando websocket e mostrando uma possível forma de interação entre cliente e servidor que ainda hoje é bastante usada e explorando um pouco do seu funcionamento.

## Requisitos funcionais
O mínimo que se espera é que estejam realmente funcionando de acordo com as especificações, se não estiverem, volte duas casas e refaça o projeto.

Uma forma simples de atingir esse objetivo é criar um arquivo "index.js" onde deve ser criada uma lista de sugestões antes de qualquer coisa no servidor. Depois usar o express para criar a rota "/" essa rota deve retornar um arquivo "index.html" que conterá o html com o formulário a lista e o javascript com o socket.IO.

No arquivo "index.js", também é preciso ter as funções do socket.IO no servidor. São necessárias duas funções, uma que retorne uma lista de sugestões com base na página informada e outra que adiciona uma sugestão à lista.

No arquivo "index.html", que é responsável pelo cliente, devem ser feitas as chamadas das funções que emitem os eventos em seus momentos específicos. Para criar o item, é preciso que se emita o evento de criação no envio do formulário impedindo que seja feita a ação padrão. 

Para a listagem, o evento que lista deve ser chamado em duas ocasiões, ao se mudar de página da listagem e ao se acessar a página. Conforme especificado nos requisitos funcionais, é importante que a página atual esteja sendo exibida na URL da página como parâmetro e, que esse parâmetro seja mudado a cada mudança de página e que seja usado para buscar a lista mesmo no primeiro acesso (sendo o padrão a página 1, caso não tenha sido informado nada), então é necessário atualizar sempre a url usando a função "pushState" e buscar usando o "window.location.search" e o objeto "URLSearchParams".

## Requisitos não funcionais
- É necessário um arquivo de documentação ensinando como rodar o projeto.
    - Baixar as dependências com "npm install" na pasta do projeto
    - Rodar o comando "node index.js"

- É necessária uma documentação de todas as requisições feitas e eventos websocket enviados pelo navegador no processo de acessar a página inicial, enviar a sugestão e exibir o novo item na lista.
        - É feita a requisição GET para a rota "/" que traz o formulário.
        - Ao enviar o formulário é impedida a ação padrão, o que evita a mudança de página e é emitido um evento para criar com a sugestão.

- É necessária uma documentação de todas as requisições feitas e eventos websocket enviados pelo navegador no processo de mudar a página da listagem.
        - É feita a requisição GET para a rota "/" que traz o formulário.
        - É emitido um evento em que é feita a busca das sugestões da página inicial
        - Ao mudar a página, é emitido um evento de listar as sugestões da nova página
