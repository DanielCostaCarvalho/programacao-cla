# Criação de um site simples usando websocket DESAFIO
## Descrição
Nesse desafio, você precisa criar um site simples usando JavaScript e as bibliotecas express e Socket.IO. O site precisa ter as mesmas funcionalidades do desafio anterior, porém apenas a rota "/" deve ser servida com uma requisição GET, todas as outras consultas devem ser feitas através de websocket.

Não é necessário se preocupar com o layout, CSS, ou em como será exibido o formulário, é necessário apenar que seja exibido e que possa ser usado. O mesmo vale para a listagem de sugestões.

## Requisitos funcionais
- O site precisa de uma rota "/" que deve conter uma lista paginada de sugestões e um formulário com um input de sugestão e um botão para enviar. 
    - No estado inicial o servidor já deve conter em memória uma lista de ao menos 3 páginas de sugestões.
    - Ao mudar de página na listagem, deve ser emitido um evento de mudança de página via websocket que o servidor deve responder com as sugestões da nova página.
    - Ao submeter o formulário, a sugestão preenchida deve ser enviada através de um evento via websocket. Ao receber esse evento, o servidor deve adicionar esse evento a listagem em memória e retornar uma mensagem de sucesso.

- O site não pode fazer nenhum redirecionamento, mudança de página, ou atualização. Ele deve apenas fazer enviar os eventos via websocket.

- A pagina da listagem deve ser exibida na url como parâmetro e, ao fazer uma requisição para a rota "/" usando esse parâmetro, o servidor deve enviar a resposta com a listagem na página informada.

## Requisitos não funcionais
- É necessário um arquivo de documentação ensinando como rodar o projeto.
- É necessária uma documentação de todas as requisições feitas e eventos websocket enviados pelo navegador no processo de acessar a página inicial, enviar a sugestão e exibir o novo item na lista.
- É necessária uma documentação de todas as requisições feitas e eventos websocket enviados pelo navegador no processo de mudar a página da listagem.
