# Criação de um site simples usando AJAX CORREÇÃO
## Descrição
Esse foi um desafio aberto a várias formas de se resolver os requisitos funcionais, então serão apontados caminhos, mas outros não apontados aqui são possíveis.

O objetivo do desafio foi mostrar na prática o funcionamento de um site usando as primeira formas de interação entre cliente e servidor abrindo portas para o conhecimento da evolução da web e do funcionamento de sistemas legado.

## Requisitos funcionais
O mínimo que se espera é que estejam realmente funcionando de acordo com as especificações, se não estiverem, volte duas casas e refaça o projeto.

Uma forma simples de atingir esse objetivo é criar um arquivo "index.php" em uma pasta "public" que busque da variável "$_SERVER" o "REQUEST_METHOD" e o "REQUEST_URI" e faça o gerenciamento de rotas. Nesse arquivos, a depender da combinação ele retornaria o "include" de um outro arquivo fora dessa pasta que retorne a página com a função de cada rota. Caso não encontre a rota, retorne um arquivo padrão (não foi solicitado nos requisitos funcionais, mas é bom).

A rota "/" deve retornar um arquivo HTML simples, mas, dessa vez, contendo também um script javascript que impedirá o comportamento padrão de submeter o formulário para fazer uma requisição por si só enviando os dados referentes a esse formulário e gerenciando o retorno para exibir uma mensagem dinamicamente de acordo com a resposta do servidor.

Dentro do arquivo referente à rota de POST da sugestão, é preciso montar a resposta em formato JSON e enviá-la, possivelmente vai ser usada a função 'json_encode' para converter um "array" do PHP em JSON. Também é interessante definir o tipo de conteúdo como JSON com a função 'header' e definir um status code com a função 'http_response_code'.
## Requisitos não funcionais
- É necessário um arquivo de documentação ensinando como rodar o projeto.
    * Usando o PHP built-in server
        - Entra na pasta publica do projeto e executa o seguinte comando
            - php -S {ip}:{porta}

- É necessária uma documentação de todas as requisições feitas pelo navegador no processo de acessar a página inicial e enviar a sugestão. Também é preciso documentar cada mudança de página.
    * A sequência de requisições é a seguinte:
        - É feita a requisição GET para a rota "/" que traz o formulário.
        - Ao enviar o formulário é feita a requisição POST para a rota "/sugestoes", porém, como é impedida a ação padrão, não é feita mudança de página
