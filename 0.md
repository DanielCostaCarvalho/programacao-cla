# Programação via CLA
## Princípios
A ideia é que você melhore por seguir alguns desafios. Muitas desses desafios consistem basicamente em "reinventar a roda", mas como a roda já existe, é provável que você já encontre muito facilmente receitas prontas para fazer "a roda". Para um melhor aproveitamento e desenvolvimento, é extremamente recomendado que você não use de respostas prontas. Então a recomendação é que você não tente buscar diretamente no google, ou use de IAs, mesmo para auto-complete, porque é provável que você já encontre uma resposta pronta e isso vai estragar a sua experiência.

## Organização
O programa é pensado para desafios semanais a serem resolvidos juntando isso com a ideia de possibilidade de escolha da parte do desafiado, por isso ao final da maioria dos arquivos existirá uma referência para outros. Para maior aproveitamento do programa, recomenda-se que não se acesse nenhum arquivo além dos que foram escolhidos. Muitas vezes serão feitas escolhas não totalmente informadas (você não terá conhecimento dos detalhes da sua escolha), mas isso é parte do programa e foi pensado para ser assim.

Ao finalizar um desafio, você poderá escolher qualquer outro desafio disponível em algum dos arquivos que já acessou.

Os desafios serão nomeados de acordo com um dos caminhos para se chegar até ele.

A ideia é que os desafios sejam semanais porque a consistência é mais importante do que a empolgação de fazer tudo de uma vez, então é altamente recomendado que se siga esse cronograma semanal.
