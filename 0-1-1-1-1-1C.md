# Criação de um site simples com autenticação CORREÇÃO
## Descrição
Esse foi um desafio aberto a várias formas de se resolver os requisitos funcionais, então serão apontados caminhos, mas outros não apontados aqui são possíveis, inclusive usando de outras tecnologias.

O objetivo do desafio foi mostrar na prática o funcionamento de um site com o básico de autenticação de forma simples, mas também usando de tecnologias que também até hoje são bastante usadas.

## Requisitos funcionais
O mínimo que se espera é que estejam realmente funcionando de acordo com as especificações, se não estiverem, volte duas casas e refaça o projeto.

Uma forma simples de atingir esse objetivo é criar um arquivo "index.php" em uma pasta "public" que busque da variável "$_SERVER" o "REQUEST_METHOD" e o "REQUEST_URI", também buscaria da variável "$_GET" os query params e da variável "$_COOKIE" os dados do cookie e faça o gerenciamento de rotas. Nesse arquivos, a depender da combinação ele retornaria o "include" de um outro arquivo fora dessa pasta que retorne a página com a função de cada rota. A depender da rota, é necessário fazer uma verificação e, se a chave não bater, retornar o arquivo de erro. Caso não encontre a rota, retorne um arquivo padrão (não foi solicitado nos requisitos funcionais, mas é bom). Repetindo basicamente o mesmo de desafios anteriores, com algumas coisas extras.

Para a rota que adiciona o cookie, é necessário chamar a função "setcookie" antes de retornar a página.

## Requisitos não funcionais
- É necessário um arquivo de documentação ensinando como rodar o projeto.
    * Usando o PHP built-in server
        - Entra na pasta publica do projeto e executa o seguinte comando
            - php -S {ip}:{porta}
- É necessária uma documentação de todas as requisições feitas pelo navegador no processo de acessar a página inicial passar pela página de query e depois pela de cookie. Também é preciso documentar cada mudança de página.
    * A sequência de requisições é a seguinte:
        - É feita a requisição GET para a rota "/" que traz o formulário.
        - Ao enviar o formulário é feita a requisição GET para a rota "/restrita-query" com a chave como parâmetro e aqui é definido o cookie por parte do servidor. Essa página contém um link.
        - Ao clicar no link, é feita a requisição GET para a rota "/restrita-cookie" já tendo o cookie definido.
    * Em todos esses momentos são feitas trocas de página.
